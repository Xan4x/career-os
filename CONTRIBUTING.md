# Contributing

## Philosophy

Career OS follows the Single Source of Truth principle.

Information must exist only once.

---

# Where should information go?

Professional identity

→ person/

Employer

→ companies/

Customer

→ customers/

Achievements

→ projects/

Knowledge

→ domains/

Generated document

→ deliverables/

---

# Project Workflow

1. Update Career OS.
2. Review.
3. Commit.
4. Regenerate deliverables.

---

# Creating a Project

Every project should describe:

- Context
- Problem
- Responsibilities
- Architecture
- Decisions
- Implementation
- Results
- Technologies

---

# Creating a Domain

A domain describes knowledge.

Never describe a project.

Reference projects when necessary.

---

# Review Checklist

- No duplicated information
- Correct cross references
- Consistent naming
- Markdown formatting
- Technologies linked to domains
